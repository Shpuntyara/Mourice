# Mourice — правила для Claude Code

## Obsidian vault — ОБЯЗАТЕЛЬНО при каждой сессии

Vault: `C:/Users/sashk/Documents/Obsidian Vault/Mourice/`

### ЧИТАТЬ перед началом работы
Перед любой реализацией или принятием архитектурного решения — **прочитай** соответствующую заметку в vault:
- Общий обзор: `Mourice.md`, `Сводный план.md`
- Дорожная карта и статус фаз: `Дорожная карта/`
- Архитектура: `Архитектура/`
- Дизайн-решения: `Дизайн/`
- Текущий прогресс и решения: `Дневник разработки.md`

Vault — **источник истины** над README и docs/. Если vault и репо расходятся — vault побеждает.

### ПИСАТЬ после каждого значимого изменения
После каждого PR / значимой фичи / важного решения — **дозапиши** в `Дневник разработки.md`:
- Что сделано и почему
- Какое решение принято (и почему именно такое)
- Что дальше

Формат: `## YYYY-MM-DD — Заголовок` в конец файла (перед комментарием `<!-- Новые записи -->`).

Не откладывай запись «на потом» — пиши сразу после merge или после того как убедился что изменение работает.

### ОБНОВЛЯТЬ при изменении решений
Если меняется архитектурное решение (модель, стек, подход) — обнови соответствующую заметку в `Архитектура/` или `Дизайн/`.

## Процесс разработки

Issues → ветка → PR → зелёный CI → merge в main. Никаких прямых пушей в main.

## Стек и конвенции

- Python, uv, ruff, mypy --strict
- Telegram: aiogram (не python-telegram-bot)
- TTS: XTTS v2 через persistent daemon (порт 5199), `.venv-xtts`
- LLM: qwen2.5:7b (default), Ollama local
- Тесты: `uv run pytest` с extras `--extra dev --extra voice --extra telegram`
